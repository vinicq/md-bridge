00 · Read me first

Five rules every new feature must respect.

These are not opinions — they're what makes md-bridge md-bridge. Any contribution that breaks one is a different product, not a feature.

1 · Deterministic

Same input, same output, every run. No AI in the conversion pipeline — pure heuristics. Features can help the user choose, but the conversion itself stays reproducible.

2 · Local-first

No external API calls during conversion. Temp files removed before the response goes out. Anything new that needs the network has to be opt-in and explicit.

3 · No persistence

No accounts, no database, no telemetry. User state lives in localStorage or the URL — that's the only memory we're allowed.

4 · Heuristic-first

Every new format pair (DOCX, EPUB, RTF) follows the same vendored-skill pattern under packages/. The route layer stays thin.

5 · Editorial UI

Fraunces + IBM Plex. Cream + ink + a single red accent. No purple gradients, no Inter, no glassy cards. Hierarchy over decoration.

02 · Roadmap

Eight features. Eight issues. One coherent system.

Ordered by the user's explicit ask first (theme picker, custom CSS, future formats, languages), then by what unlocks the rest. Each card below carries a type, a scope, and a complexity hint so contributors can self-select.

Page size, margins, TOC, page numbers, header/footer text. Per file or per batch.
F4 · platform USER ASK

More formats: DOCX · EPUB · HTML

F5 · i18n USER ASK

Language workshop

Conversion presets

F7 · UX local-only

Local conversion history

File names + sizes + result blob URLs in localStorage. Re-download a recent result without re-running.
F8 · settings polish

Preferences page

Default theme, default locale, accent color override, "no telemetry" affirmation badge.

## Problem

Closing the tab loses the blob URL and the user re-uploads the same file just to grab the .md again. The project's "no persistence" rule applies to the **server**; the browser is fair game.

## Proposal

A "Recent" panel under the dropzone keeps a list of conversions in `localStorage`:
- File name, size, format pair, options digest, timestamp, last blob URL.
- Blob URLs alive while the tab is — after 24h or page refresh the row is "expired" and offers Re-run.

## Acceptance criteria

- [ ] Storage key: `md-bridge:history`. Max 20 entries, FIFO eviction.
- [ ] Each row: name, size, format, settings summary, age, status badge, Re-download (if alive), Re-run.
- [ ] "Clear all" on the panel header.
- [ ] Privacy paragraph below the list (translated EN / PT / ES).
- [ ] About page gets a new line: "No server persistence. Optional local browser history."
- [ ] Tests: FIFO cap, expiry detection, serialization round-trip.

## Problem

By F7 there are 10+ `localStorage` keys: locale, theme, preset list, history, custom CSS, options digest. The user has no single place to inspect or reset them.

## Proposal

One page at `/preferences` grouped in three sections: **Defaults**, **UI**, **Privacy**. Each row links to the feature that owns it. "Reset all" clears the `md-bridge:*` namespace.

## Acceptance criteria

- [ ] All settings persisted under `md-bridge:prefs` (single key, merged on read).
- [ ] Accent colour writes a CSS variable override on `:root`.
- [ ] Reduce-motion respects `prefers-reduced-motion` by default; manual toggle overrides.
- [ ] "No telemetry" badge points at `scripts/audit-deps.py` output committed in CI.
- [ ] Tests: read/write round-trip, reset clears every `md-bridge:*` key.

03 · Appendix

Issue hygiene — how to file these as a maintainer.

Short checklist to keep this batch of issues tidy as community PRs land. Reuses the existing labels and templates.

1 · One issue per F-number

Paste the markdown card into .github/ISSUE_TEMPLATE/feature_request.md. Title prefix matches the bracket convention already in use (e.g. [feature], [i18n], [platform]).

2 · Link the doc

Every issue body links back to this file under docs/design/Design Thinking.html. That way a contributor can see the mock and the rationale before writing a line of code.

3 · Suggested order

F3 (Options) → F1 (Themes) → F2 (Custom CSS) → F6 (Presets) → F7 (History) → F8 (Prefs) → F4 (Formats) → F5 (i18n).

Front-load the dependency root (Options panel) so everything else has somewhere to attach.
